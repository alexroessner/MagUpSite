Design Philosophy

This site is generated from two inputs: a PDF document (content) and a reference URL (design). The design system merges both streams into a cohesive visual identity.

  • Content-first: Design serves the content, not the other way around.
  • Accessible by default: WCAG 2.1 AA minimum. Underlined links, visible focus states, sufficient contrast.
  • Zero client-side JavaScript: Only a hamburger menu toggle. Everything else is HTML + CSS.
  • System fonts for body: Zero load cost, excellent screen readability.
  • Responsive from mobile up: Every section works at 320px before scaling to desktop.

Accessibility

Target: WCAG 2.1 AA compliance on all pages.

  • Skip link: Hidden until focused, jumps to #main.
  • Links: Permanently underlined (WCAG 1.4.1 — not just on hover).
  • Focus states: Visible on all interactive elements.
  • Color contrast: All text-on-background combinations meet AA ratios.
  • Navigation: aria-current="page" on active items, aria-expanded on hamburger.

The Grayscale Test

View the site with color removed and ask: "Can I still tell what's interactive?" Links should be identifiable by their underline, buttons by their shape, and focus states by their outline — not just by color.
